# 🍌 BananaParser - Умный парсер вирусных мемов из Telegram

BananaParser — это масштабируемый автономный движок для Telegram, который находит самые вирусные мемы и медиа-контент до того, как они станут мейнстримом. Парсер работает 24/7, собирает аналитику, очищает ваш канал от ботов и фильтрует дубликаты с помощью продвинутых алгоритмов машинного зрения.

## 🌟 Ключевые возможности

1. **Математический поиск вирусности (CFS & RVI)**
   Алгоритм не просто смотрит на лайки. Он классифицирует каналы-доноры по кластерам (от Nano до Bridge) и вычисляет **Relative Viral Index (RVI)** — аномалию поста относительно скользящей средней (EMA) вовлеченности конкретного канала. Вы получаете только те посты, которые "выстрелили".

2. **Абсолютный Анти-Баян (dHash + pHash)**
   Больше никаких повторяющихся мемов.
   * **Глобальный архив (seen_archive.js):** Использует 64-битный разностный хеш (dHash) для постоянного хранения базы увиденных изображений. dHash устойчив к изменению яркости и вотермаркам.
   * **Локальный кеш (dedup.js):** Использует сверхточный 15376-битный (124x124) перцептивный хеш (pHash) для бескомпромиссной фильтрации ложных дубликатов в реальном времени.

3. **Глубокая очистка от ботнетов (Deep Cleaner)**
   Скрипт `cleaner_deep.js` может прочесать весь список участников вашего канала, обходя стандартный лимит Telegram в 200 пользователей через алфавитный перебор. Он автоматически находит индусов, спам-аккаунты и "пустышек", удаляя их. Встроен обработчик лимитов Telegram API (`FLOOD_WAIT`), поэтому скрипт работает полностью автономно даже при банах десятков тысяч ботов.

4. **Автоматическое обнаружение новых каналов**
   Модуль `_autodiscover_block.js` анализирует пересылаемые посты (репосты) и автоматически собирает базу новых потенциальных доноров контента.

5. **Веб-панель управления**
   Встроенный локальный `admin_server.js` дает доступ к дашборду для удобного управления базой каналов, кластерами и мониторинга аналитики прогонов.

## ⚙️ Формула вирусности

```text
VI  = views + (reactions × 5) + (replies × 3)
RVI = VI / EMA_канала          (аномалия относительно нормы)
CFS = RVI × SizeModifier × FreshnessPenalty
```
Посты с наивысшим CFS (Composite Final Score) отбираются для публикации в целевой канал.

## 🚀 Быстрый старт

### Что нужно
- **Linux сервер** (Ubuntu 20.04+) с Node.js 18+
- **Telegram аккаунт** (не бот!) для парсинга и пересылки.
- **Telegram бот** для уведомлений и публикации краткой статистики.
- **API ключи** с [my.telegram.org](https://my.telegram.org)

### 1. Подготовка и установка

1. Склонируйте репозиторий:
```bash
git clone https://github.com/fsbtactic-code/bnntg-parser.git
cd bnntg-parser/smart_meme_parser
npm install
```

2. Настройте конфигурацию:
```bash
cp config.example.json config.json
cp .env.example .env
```

Отредактируйте `config.json` и добавьте ваши каналы:
```json
{
    "apiId": 12345678,
    "apiHash": "твой_api_hash",
    "targetChannels": [ "донор_1", "донор_2" ],
    "destinationChannel": "@твой_канал",
    "hoursToCheck": 0.5,
    "maxMemesToForward": 30
}
```

В `.env` укажите токен вашего бота:
```env
BOT_TOKEN=1234567890:AAFxxxx...
BOT_CHANNEL_ID=-100твой_chat_id
```

### 2. Авторизация
Сгенерируйте сессию Telegram-клиента (потребуется код подтверждения из Telegram):
```bash
node gen_session.js
```
*(Сессия сохранится в файл `session.txt`. Никому его не передавайте!)*

### 3. Запуск

Парсер работает в цикле (cron-стиль). Рекомендуется запускать через PM2:
```bash
npm install -g pm2
pm2 start index.js --name "banana-parser"
pm2 save
```

### 4. Запуск очистки от ботов
Если на канал накрутили ботов, запустите фоновый клинер:
```bash
pm2 start cleaner_deep.js --name "bot-cleaner"
```

### 5. Админ-панель (опционально)
Запустите веб-интерфейс для управления донорами:
```bash
pm2 start admin_server.js --name "banana-admin"
```
Панель будет доступна по адресу `http://localhost:8333`.

## ❓ FAQ

**Q: Скрипт выдал `FLOOD_WAIT_X`. Что делать?**  
A: Ничего. В BananaParser встроена система авто-ожидания (Rate Limit Handling). Парсер уснет на указанное Телеграмом количество секунд и продолжит работу автоматически.

**Q: Парсер пересылает чужой пост. Можно ли публиковать без указания автора?**  
A: Пересылка работает через `forwardMessages`. Чтобы скрыть автора, в новых версиях API необходимо выставлять флаг `dropAuthor: true` (если канал позволяет).

**Q: База хешей `seen_archive.bin` растет, что делать?**  
A: База оптимизирована (8 байт на запись). 100,000 сохраненных постов занимают менее 1 МБ памяти. Вмешательство требуется крайне редко.
